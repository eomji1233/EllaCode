HTML4 페이지 구조

HTML4에서 페이지 구조 div, span 태그를 이용, 구역을 설정하고 id값을 부여하여 구분함.
* 단점
1. 사용자마다 id값을 다르게 부여하여 구조를 이해하는 어려움이 있고 유지보수에도 어려움이 있음
2. 검색엔진이 웹 문서를 확인할 때도 메뉴, 본문 구별이 어려움

HTML5 페이지 구조

시맨틱 태그를 사용함
1. 특정 기능에 마는 태그를 사용하여 페이지 구조를 구분
2. 페이지 구조를 쉽게 파악할 수 있고, 정확한 정보를 검색할 수 있음.

* header : 특정 부분의 머리말로 주로 검색어(form), 메뉴(nav)를 넣음
- nav : 다른 사이트 페이지로 이동하는 태그를 모아 놓은 태그, 특정 태그에 종속되지 않고 어느 곳에서나 사용할 수 있음

* main : 핵심 컨텐츠를 담는 태그, 웹 문서마다 다르게 보여주는 내용으로 구성, 웹 문서에서 한 번만 사용
* section : 웹 문서에서 컨텐츠가 들어가는 영역, 컨텐츠를 주제 별로 묶을 때 사용, section 태그 안에 section 태그 사용 가능
* article : 독립적인 컨텐츠를 담는 태그, 여러 개 사용할 수 있고, section 태그를 넣을 수도 있음. 웹 페이지의 내용이 들어가는 영역, 이 영역은 다른 곳으로 배포하거나 재사용이 가능
* aside : '사이드 바' 라고 불리며 본문 외에 기타 내용을 담고 있는 영역, 주로 광고를 달거나 링크 모음 등을 표현
* footer : 웹 페이지의 맨 아래쪽에 위치하며, 회사소개, 저작권, 연락처 등의 정보 표시
- address : 주소를 표시할 때 사용
